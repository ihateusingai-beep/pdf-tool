# 📚 專業 PDF 教職員工作站

專為特殊教育 (SEN) 教師與行政人員設計的**純前端、零伺服器** PDF 處理工具。所有操作均在本地端完成，確保學生敏感個資不會外洩。

## 1. 功能概覽

本系統提供四種作業模式，透過頂部導航列切換：

| 模式 | 圖示 | 說明 |
|------|------|------|
| 📄 合併與重組 | 藍靛色 | 將多個 PDF 合併為一個，支援拖曳排序與插入空白頁 |
| ✂️ 提取與分割 | 翡翠綠 | 將 PDF 拆分為多個檔案，或展開為 ZIP 壓縮包 |
| 📝 Word 轉 PDF | 海藍色 | 將 Word 文件 (.docx) 批次轉換為 PDF |
| ⚙️ 批次加工作業 | 琥珀色 | 對多個 PDF 批次加工（加頁碼、空白頁）並打包下載 |

## 2. 操作指南

### 2.1 合併與重組

```
① 上傳 PDF → ② 拖曳排序 → ③ 新增空白頁（可選）→ ④ 下載
```

**步驟說明：**

1. **上傳檔案**：將 PDF 檔案拖曳至虛線框內，或點擊框體選擇檔案
2. **拖曳排序**：按住縮圖並拖曳至目標位置，松開滑鼠完成排序
3. **插入空白頁**：點擊「插入空白頁」按鈕，在列表中加入空白頁面
4. **合併下載**：確認順序無誤後，點擊「合併下載」取得整合後的 PDF

> 💡 **提示**：系統會自動依據檔名中的數字進行智能排序（例：練習_2 會排在練習_10 之前）

---

### 2.2 提取與分割

```
① 上傳 PDF → ② 點選頁面預覽 → ③ 選擇輸出方式 → ④ 下載
```

**步驟說明：**

1. **上傳檔案**：將 PDF 拖曳至虛線框內
2. **選擇頁面**：點擊要提取的頁面縮圖（選中會顯示綠色框線）
3. **選擇輸出方式**：
   - **擷取選中頁面**：將選中的頁面合併為一個 PDF 檔案
   - **展開為 ZIP**：將每個選中的頁面分別儲存為獨立的 PDF，並包裝為 ZIP 壓縮包

---

### 2.3 Word 轉 PDF

```
① 上傳 .docx → ② 批次轉換 → ③ 下載 ZIP
```

**步驟說明：**

1. **上傳檔案**：將 Word 檔案 (.docx) 拖曳至虛線框內
2. **批次轉換**：點擊「批次轉換」按鈕
3. **下載**：系統會自動將所有轉換後的 PDF 包裝為 ZIP 壓縮包下載

> ⚠️ **限制**：本功能支援 .docx 格式，不支援舊版 .doc 格式。請確認檔案可使用 Word 開啟。

---

### 2.4 批次加工作業

```
① 選擇加工項目 → ② 上傳 PDF → ③ 批次下載
```

**步驟說明：**

1. **選擇加工項目**：
   - ☑️ 加入頁碼：在每頁右下角加入頁碼
   - ☐ 插入空白頁：在奇數頁後插入空白頁
2. **上傳檔案**：將多個 PDF 拖曳至虛線框內
3. **批次下載**：系統會自動加工並包裝為 ZIP 壓縮包

## 3. 部署方式

本系統無需安裝任何後端服務，請根據使用情境選擇以下方式：

### 3.1 本機執行（推薦給個人教師）

1. 將 `index.html` 檔案複製到本機
2. 以 Chrome、Edge 或 Safari 瀏覽器直接開啟該檔案
3. 即可開始使用

> 📌 **瀏覽器相容性**：建議使用最新版本的 Chrome、Edge 或 Safari

---

### 3.2 內網部署（推薦給學校 IT 部門）

1. 將整個資料夾上傳至學校檔案伺服器
2. 設定路徑，例如：`http://school-server/pdf-tool/`
3. 教師透過學校內網即可存取

**SharePoint 部署範例：**
1. 將資料夾上傳至 SharePoint 文件庫
2. 點擊 `index.html` → 複製連結
3. 分享給校內教職員

---

### 3.3 雲端部署（任何靜態托管服務）

將資料夾上傳至以下任一免費服務：

| 服務 | 網址 | 設定方式 |
|------|------|----------|
| GitHub Pages | github.com | 推送至 `gh-pages` 分支 |
| Netlify | netlify.com | 拖曳資料夾即可 |
| Vercel | vercel.com | 連接 Git 倉庫或拖曳上傳 |

```
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ │
└───────┼────────────┼───────────┼───────────┼──────┘
        │            │           │           │
┌───────▼───────────▼───────────▼───────────▼──────┐
│                  PDF 引擎層                        │
│  ┌───────────────────┐  ┌───────────────────┐   │
│  │     pdf-lib       │  │      pdf.js       │   │
│  │  (操作/修改/生成) │  │   (渲染/預覽)     │   │
│  └───────────────────┘  └───────────────────┘   │
│  ┌───────────────────┐  ┌───────────────────┐   │
│  │    mammoth.js    │  │   html2pdf.js    │   │
│  │  (docx → HTML)   │  │ (HTML → PDF，含  │   │
│  └───────────────────┘  │  html2canvas)    │   │
│                          └───────────────────┘   │
└──────────────────────────────────────────────────┘
```

### 4.2 錯誤處理矩陣

| 情境 | 徵兆 | 解決方式 |
|------|------|----------|
| PDF 結構損壞 | 檔案無法讀取 | 請重新上傳，或使用 Word/Adobe 重新儲存該 PDF |
| 檔案過大 (>50MB) | 處理時間過長或當機 | 建議使用「提取與分割」模式分段處理 |
| DOCX 格式不符 | 無法轉換 | 請確認檔案為 .docx 格式，並使用 Word 重新儲存 |

## 5. 術語表

| 術語 | 說明 |
|------|------|
| **展開為 ZIP** | 將選中的頁面分別儲存為獨立 PDF，並包裝為 ZIP 壓縮包下載 |
| **檔名模板** | 分割匯出時支援 `{n}`（頁碼）、`{date}`（日期）變數 |
| **視覺化預覽排序** | 透過縮圖預覽而非檔名進行拖曳排序 |
| **批次加工** | 對多個檔案同時套用相同的處理（加頁碼、空白頁） |
| **智能排序** | 自動辨識檔名中的數字（例如：練習_2 會排在練習_10 之前） |

## 安全驗證方式

教師可透過以下方式自行確認「零外流」聲明：

1. **Network 標籤**：按下 F12 開發者工具，切到 Network 分頁，進行任何操作後確認無外發請求
2. **離線測試**：斷開網路後，確認所有功能（除 CDN 載入階段）仍正常運作
3. **暫存清除**：上傳測試 PDF，處理完畢後檢查瀏覽器暫存區已清除

```bash
cd tests

npm test           # T1 (page load) + T2 (merge mode output) + T11 (A5 metadata strip)
```
